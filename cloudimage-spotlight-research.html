00 TL;DR

Build it. The idea is well-scoped, technically elegant, and solves a real daily pain for the Scaleflex team. The key insight: this is not another demo tool competing with Arcade or Storylane — it is a CDN-native embeddable player that lives inside an authoring platform you already have. The moat is Cloudimage's image transformation URL API used natively as the visual engine. No one else can do this the same way.

01 Market Landscape

The interactive demo / product tour market is mature, crowded, and expensive. Understanding it precisely defines where your plugin can win.

Category 1 — Live DOM Walkthroughs

Libraries like Shepherd.js, Driver.js, Intro.js, TourGuide.js. They overlay tooltips on actual running DOM elements. Free/open-source, widely used. Problem: break every time your UI updates. Not your competition — different product.

Category 2 — Screenshot-Based Demo Platforms

This is where the direct comparables live. These are SaaS tools that capture screenshots of your app and let you build guided flows from them.

Tool Model Price Authoring Deployment
Arcade Screenshot + screen recording ~$32/mo free tier available Chrome extension capture Their hosted iframe embed
Storylane Screenshot + HTML + video $40–$100/user/mo Browser extension + AI Their hosted embed
Navattic HTML-first $500–$1,000+/mo Chrome extension HTML capture Their hosted embed
Supademo Screenshot-based Free tier; ~$27/mo paid Chrome extension Their hosted embed
Walnut Sales-focused HTML $9,200+/year minimum Browser extension Their hosted embed

The universal pattern: Every single competitor locks you into their hosting, their embed URL, their analytics, their pricing seat model. You cannot self-host the player. You cannot bring your own CDN. You cannot control the image pipeline. This is the structural gap.

Category 3 — Image Annotation Libraries

Open-source tools like Annotorious, marker.js 2, Annotator.js. These provide the raw annotation primitives (draw regions, attach notes) but have no concept of scenes, sequencing, transitions, or CDN-backed image operations. They're building blocks, not products.

03 The CDN Advantage

This is the most technically interesting and underappreciated aspect of the idea. Every visual operation in the experience can be a native Cloudimage URL transformation — delivered, cached, and optimized by the CDN. Not CSS. Not canvas. Actual server-side image operations.

Visual Effect Competitor approach Cloudimage approach
Zoom to region CSS transform: scale() on a div ?func=crop&tl_x=0.2&tl_y=0.1&br_x=0.6&br_y=0.5
Blur outside region CSS backdrop-filter or canvas overlay Deliver two images: sharp crop + blurred full version layered
Responsive delivery Fixed pixel-size screenshots, no srcset Automatic width/device-pixel-ratio optimization
Format optimization Whatever format you uploaded Auto WebP/AVIF, compressed to optimal quality
Grayscale "fade" effect CSS filter: grayscale() ?grey=1 delivered server-side

When you zoom to a region, you are requesting a pre-cropped image URL from the CDN. That image is cached globally. On the second request from any user anywhere, it loads from edge. A CSS zoom has no such benefit — it's always the full image, scaled awkwardly. This is a genuine quality advantage, not just a talking point.

Key Cloudimage params used in the plugin

// Positionable crop — "zoom into this region of the screenshot" ?func=crop&tl_x=0.15&tl_y=0.20&br_x=0.55&br_y=0.65&w=1200 // Blur the whole image — "soften the background before spotlighting" ?blur=8&w=1200 // Full scene image — sharp, optimized, right size ?w=1200&q=85 // Grayscale version for "inactive scene" previews in nav ?grey=1&w=240

05 Scene Data Model

A well-designed JSON config is the contract between Spotlight (authoring) and the plugin (player). It should be human-readable, portable, and extensible.

{ "version": "1.0", "ciToken": "your-cloudimage-token", "title": "Scaleflex DAM — Asset Library Tour", "settings": { "transition": "fade", // fade | slide | zoom "autoplay": false, "showProgress": true, "allowKeyboard": true }, "scenes": [ { "id": "scene-1", "image": "https://demo.scaleflex.com/screenshots/asset-library.jpg", "title": "Asset Library Overview", "description": "Your central hub for all media assets.", "region": null, // null = show full image "mask": false }, { "id": "scene-2", "image": "https://demo.scaleflex.com/screenshots/asset-library.jpg", "title": "Smart Tagging", "description": "AI automatically tags assets on upload.", "region": { "tl_x": 0.62, // normalized 0–1 coordinates "tl_y": 0.08, "br_x": 0.98, "br_y": 0.44, "shape": "rect" // rect | ellipse | polygon (future) }, "mask": true, // dim everything outside the region "zoom": "cdn", // "cdn" = use Cloudimage crop URL | "css" = CSS transform "annotation": { "position": "bottom-right", "style": "card" // card | tooltip | inline } } ] }

Normalized coordinates (0–1 range) are critical. They make the config resolution-independent — the same JSON works whether the screenshot is displayed at 800px or 1600px wide. The URL builder scales them to pixel values at request time based on container dimensions.

06 UX Patterns & Interactions

Research on top-performing product tours shows that the spotlight/mask pattern consistently outperforms plain tooltips for feature discovery because it reduces cognitive load — the user has no other place to look. Here are the key patterns to implement:

🔦 Spotlight mask

Dim everything outside the highlighted region. Best practice: use a semi-transparent dark overlay with a "cut-out" using CSS clip-path or SVG mask — not a blurred image. The sharp contrast between the lit region and dimmed background is what makes this effective.

🔍 CDN zoom

When zooming to a region, request the cropped image URL from Cloudimage. Animate a smooth transition from the full screenshot to the zoomed crop using opacity/scale CSS — the new image replaces the old, giving a "pan and zoom" feel without any CSS transform jank.

💬 Annotation card

A floating card or tooltip attached to the highlighted region. Should support: title, description, step counter (2 of 5), previous/next controls. Keep it compact. Research shows users skip long annotation text. Aim for <20 words per step.

⌨️ Keyboard nav

Arrow keys for prev/next, Escape to close. Non-negotiable for accessibility. Also expected by power users in demo contexts (presentations, screenshares). A pulse animation on the highlighted region helps the eye find it, but should stop after 1–2 cycles.

🎬 Scene filmstrip

Optional bottom navigation strip showing thumbnails of all scenes. Uses grayscale Cloudimage URLs for inactive scenes (?grey=1&w=80). Gives users a sense of the full experience length and lets them jump non-linearly.

📱 Responsive layout

On mobile, annotation cards switch from floating overlay to a bottom sheet pattern. The image fills the screen width; the card slides up from below. Cloudimage delivers correctly sized images automatically — this is free behavior.

Transition best practices (from research)

Leading demo tools use fade transitions as the default — they work universally, don't cause motion sickness, and keep the image as the focal point. Slide transitions work well for sequential workflows (step 1 → step 2 → step 3). Zoom transitions are high-impact but should be optional. Implement all three; default to fade.

Key UX rule discovered from competitor reviews: always provide a "Skip" option and a step counter. Users abandon demos that feel like they have no exit. Progress visibility ("2 of 5") dramatically improves completion rates.

09 Risks & Mitigations

Screenshot staleness

Risk: Screenshots go out of date when the UI changes. Unlike DOM tours, they don't auto-update.

Mitigation: Design the authoring tool with a "re-capture" flow. Store screenshots in the DAM — versioning is then automatic. Highlight in docs that this is by design: frozen demos are consistent, not broken.

Region editor complexity

Risk: Drawing regions on screenshots in Spotlight requires a non-trivial canvas/SVG editor inside the dashboard.

Mitigation: Start with rectangle regions only (simple 4-corner drag). Ellipse and polygon shapes can come in v2. The Annotorious or marker.js libraries are solid foundations for this editor.

CDN zoom quality

Risk: If the source screenshot is low-resolution, CDN crop zoom will look blurry/pixelated.

Mitigation: Document minimum recommended screenshot resolution (1920px+). Spotlight should warn when an uploaded image is below the threshold. This is the one case where CSS zoom may look better — offer it as a fallback.

Overlay precision on responsive layouts

Risk: Region coordinates defined at 1920px width may not align correctly at 375px mobile width.

Mitigation: Use normalized 0–1 coordinates (not pixel values). The overlay engine recalculates pixel positions on every resize event. This is why the data model section recommends normalized coords — it's not cosmetic, it's architectural.

10 Build Roadmap

Phased approach that lets you ship quickly and use the plugin for real before committing to full feature depth:

01
Plugin runtime (player only)
Vanilla JS/TS plugin. Takes a JSON config. Renders scene sequence with spotlight mask overlay, annotation card, prev/next nav, keyboard support. No authoring — config is hand-written JSON. This gets you running on demo calls immediately.
~2 weeks
02
Cloudimage URL builder
The CDN-native zoom logic. Given region coordinates + container dimensions + CI token, build the correct crop URL and pre-warm it. This is the technical differentiator that makes the plugin genuinely better than CSS zoom alternatives.
~1 week
03
Spotlight authoring integration
New experience type in the Spotlight dashboard. Scene list, image upload, region drawing editor (rectangles first), annotation text editor, preview mode. Output: the JSON config. This is the biggest chunk of work but highest leverage — non-developers can now create experiences.
~4–6 weeks
04
Transition system + filmstrip nav
Fade, slide, zoom transitions. Bottom filmstrip with grayscale thumbnails. These dramatically improve the experience quality but aren't required for MVP. Ship after v1 gets real usage.
Post-v1
05
Analytics hooks + embed options
onSceneView, onComplete, onSkip callbacks for GTM/analytics integration. Embed via script tag, npm package, or React component wrapper. Generate embeddable iframe from Spotlight (same as hotspot embed).
Post-v1
06
Advanced region shapes + AI annotation
Ellipse regions, polygon support. Potential: AI-generated annotation text from screenshot + context using Anthropic/OpenAI API. You paste the screenshot, the AI drafts the step copy. Very aligned with where Storylane and Arcade are heading with AI assist.
v2

11 Verdict

The concept is sound, the timing is right, and the technical moat is real. The market has proven strong demand for screenshot-based demo experiences — every major competitor is profitable and growing. The gap is not in demand; it is in who builds them. All incumbents are SaaS businesses charging per-seat, per-month, locking content behind their CDN. A self-hostable, CDN-native, open-source player embedded in an existing authoring platform is genuinely differentiated.

The name is cloudimage-spotlight. The first milestone is the runtime player with hand-written JSON config — ship it in two weeks, use it for real on the next demo call, then build the Spotlight authoring integration once you've validated the scene model against real usage.

Internal value

Immediate — replaces static slide decks and screen recordings for Scaleflex demo calls. Sales and CS teams become power users from day one.

Client value

High potential — especially in e-commerce, SaaS, and automotive. Positioned as a free plugin that unlocks value from their existing Cloudimage subscription.

Technical differentiator

CDN-native image operations. This is the one thing no competitor can replicate without also being a CDN. It's not a feature — it's a structural advantage baked into the stack.

Ecosystem fit

Perfect. Follows existing plugin patterns, lives in Spotlight, is powered by Cloudimage. No new product surface — just a new experience type inside a platform that already exists.
